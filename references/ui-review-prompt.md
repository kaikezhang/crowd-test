# UI Review Prompt — 极致严苛评估模板

> 用途：主 agent 在生成 persona 的 feedback 评估指令时参考此模板。不是直接复制给 sub-agent，而是从中提取评估维度和严格度融入 persona 的反馈指令中。

---

你的任务是对任意移动端或网页端界面截图进行**极致严苛、全面、结构化**的评估。

评估规则：
- 零容忍：任何可能影响用户体验、品牌形象、专业度、转化率或导致用户流失的问题都必须明确指出，即使是细微感觉问题。
- 必须覆盖所有维度，不得遗漏。
- 每项评估必须给出0-10分，并详细说明扣分理由。
- 必须标注优先级：【致命】【严重】【中等】【轻微】【优化建议】。
- 输出必须严格使用指定结构，不得增加额外解释或模糊描述。
- 语言必须直接、专业、具体、可执行。

输出必须严格按照以下结构（Markdown + JSON 双格式）：

### 1. 界面概览
（一句话描述当前屏幕主要功能和视觉风格）

### 2. 详细评估

**A. 视觉美观度（Visual Appeal）** [0-10]
- 色彩搭配、品牌一致性、留白、视觉层级、现代感等

**B. 布局与排版（Layout & Typography）** [0-10]
- 响应式适配、元素对齐、字体层级、阅读舒适度等

**C. 交互可用性（Usability & Intuitiveness）** [0-10]
- 操作路径清晰度、控件明显度、反馈及时性等

**D. 潜在Bug与技术问题（Bugs & Technical Issues）** [0-10]
- 元素重叠、溢出、文字截断、点击热区、加载异常等

**E. 无障碍与包容性（Accessibility）** [0-10]
- 对比度、色盲友好、字体大小、屏幕阅读器兼容等

**F. 边缘场景对抗测试（Adversarial Scenarios）**
- 至少列出3个可能导致问题的极端场景及预期后果

### 3. 总体评分与风险等级
- 总分：[0-100]
- 风险等级：【致命缺陷】【高风险】【中风险】【低风险】【优秀】

### 4. 改进建议（按优先级排序）
1. 【致命】xxx → 具体改动建议
2. 【严重】xxx → 具体改动建议
3. ...
